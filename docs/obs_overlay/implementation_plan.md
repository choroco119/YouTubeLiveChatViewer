# OBS配信用ローカルWebサーバー（ブラウザソース用オーバーレイ）実装計画

Geminiの最新の回答をOBS等の配信ソフトに美しく透過させて取り込めるよう、アプリケーション内部に超軽量なWebサーバーを内蔵し、ブラウザソース用のHTML/CSS/JS画面を配信する機能を実装します。

## ユーザー確認が必要な事項

> [!NOTE]
> アプリ起動時にローカルポート（デフォルト: 3000）を占有します。ポート衝突を回避するため、ポート番号は設定画面から変更できるようにします。
> また、外部のWebサーバー（ApacheやNginxなど）をインストールすることなく、本アプリ単体で完結して動作します。

## 提案する変更点

### 1. 設定情報の拡張 (`src/settings.rs`)
- `Settings` 構造体に以下のフィールドを追加します。
  - `obs_server_enabled`: Webサーバーの有効化フラグ（デフォルト: `false`）
  - `obs_server_port`: サーバーが待受するポート番号（デフォルト: `3000`）
- 設定のデフォルト値関数を定義します。

### 2. Webサーバーモジュールの新規作成 (`src/web_server.rs` [NEW])
- 外部クレートへの依存を抑え、アプリを軽量に保つため、Tokioの `TcpListener` を利用した超軽量なHTTPサーバーを実装します。
- **提供するエンドポイント**：
  - `GET /` : 表示用HTML（CSS/JSを内包）を返却
  - `GET /api/response` : 最新のGeminiの回答をJSON形式で返却 (`{"response": "...", "timestamp": "..."}`)
- **表示用HTMLのデザインシステム（案）**：
  - 完全な透過背景（OBS側で自動的に背景が抜けます）。
  - フォント: Google Fontsの「Outfit」または「Noto Sans JP」を動的読み込み。
  - 吹き出し風またはネオン風の枠線（シアンとパープルのスタイリッシュなグラデーション）。
  - 文字の可読性を上げるためのほのかなドロップシャドウ。
  - **切り替えアニメーション**: JSで `/api/response` を1秒おきに監視。テキストが更新された際、古いテキストがスライドアウト/フェードアウトし、新しいテキストが滑らかにスライドイン/フェードインするアニメーション効果。

### 3. アプリケーション状態への統合 (`src/app.rs`)
- **UI（設定パネル）**：
  - 左側のスクロールエリアに「OBS配信表示連携」セクションを追加。
  - 「OBS配信用サーバーを起動」のチェックボックス。
  - 「ポート番号」の数値入力欄（DragValueまたはTextEdit）。
  - 起動中にブラウザソース用に入力すべきURL（`http://localhost:3000`）をクリップボードにコピーするボタンを併設。
- **制御ロジック**：
  - アプリ起動時、`obs_server_enabled` が `true` であれば `tokio::spawn` でWebサーバーを起動。
  - 実行中に設定が変更（有効化/無効化、ポート変更）された場合、サーバーを安全に再起動または終了する制御。
  - Geminiからの新しいレスポンスを検知した際、共有状態の変数 `latest_response` を更新。

### 4. モジュール登録 (`src/main.rs`)
- `mod web_server;` を追加。

## 検証計画

### 開発時の検証
- ブラウザから `http://localhost:3000` にアクセスし、HTMLが正しく返されること、デザインやアニメーションが効くことを確認。
- `http://localhost:3000/api/response` にアクセスし、JSONデータが正しく取得できること。
- アプリの左パネルで設定を保存し、アプリ再起動時に正しくサーバーが立ち上がることを確認。

### OBSでの検証
- OBSで「ブラウザソース」を追加し、URLに `http://localhost:3000` を設定。
- コメントを模擬送信し、Geminiの回答がOBS上のブラウザソースに透過かつフェードインアニメーションを伴って反映されるかを確認。
