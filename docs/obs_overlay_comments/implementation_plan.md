# OBSオーバーレイでの通常コメント統合表示 実装計画

OBS配信用Webサーバーで配信されるオーバーレイ画面（`http://localhost:3000`）に、Geminiの回答に加えて、YouTube Liveの通常コメントも同一画面上に統合してリアルタイム表示（スクロール追従）するための実装計画です。

## ユーザー確認事項

> [!IMPORTANT]
> **レイアウトとデザインの提案**
> 1つの画面に統合するため、画面を左右に分割する **「2カラムレイアウト」** を提案します。
> *   **左側（幅約350px）**: YouTube Liveのチャットフィード。新しいコメントが下から追加され、上にスクロールして押し出されます。
>     *   モデレーター、メンバー、配信者などのユーザー権限に応じた名前の色分けやバッジ表示をサポートします。
> *   **右側（幅約550px）**: Geminiの回答（吹き出し）。これまで通りのネオンパープルの洗練されたカードデザインで、回答が届いたときにフェード＆スライドインします。
> *   **背景**: 完全透過（OBSのソースとして重ねて配置可能）。
> 
> この構成により、配信画面の横幅に合わせてOBS上で綺麗に配置することができます。このレイアウト方針でよろしいでしょうか？

## 提案する変更内容

### 1. Webサーバーの拡張

#### [MODIFY] [web_server.rs](file:///c:/Users/kohei/.gemini/antigravity/scratch/YouTubeLiveChatViewer/src/web_server.rs)
*   `WebServerState` 構造体に、最新のコメント一覧を保持する `comments: Vec<ObsComment>` フィールドを追加します。
*   `GET /api/response` で返却する JSON ペイロードに `comments` 配列を追加します。
*   `get_overlay_html()` 内のHTML/CSS/JSをアップデートし、左右2カラムの統合デザインに変更します。
    *   **CSS**: Flexboxによる2カラム構成。通常コメント用のカードにふんわりとしたフェードインアニメーション（`animation: slideIn 0.3s ease-out`）を適用。
    *   **JS**: `/api/response` を1秒ごとにポーリングし、新しいコメントID（`id`）を検知した際に DOM に要素を追加。追加後に自動で最下部までスクロールするアニメーションを実装。DOM内のコメント数が一定数（例: 20件）を超えたら、古いものを自動削除してメモリリークを防ぎます。

### 2. アプリケーション状態管理の拡張

#### [MODIFY] [app.rs](file:///c:/Users/kohei/.gemini/antigravity/scratch/YouTubeLiveChatViewer/src/app.rs)
*   `poll_chat_events` 内で、新しいコメントを受信するたびに `web_server::ObsComment` 構造体に変換し、`web_server_state` 内の `comments` 配列に追加する処理を追加します（直近最大50件程度を保持）。
*   配信接続時（`connect` 関数）に、過去のOBS側コメント履歴をクリアする処理を追加します。

## 検証計画

### 自動テスト
- `cargo check` および `cargo test` を実行し、コンパイルエラーや退行が発生しないことを確認します。

### 手動検証
1. アプリを起動し、OBS配信連携を有効化（ポート: 3000）します。
2. ブラウザで `http://localhost:3000` にアクセスします。
3. YouTube配信に接続するか、テストログを読み込みます。
4. 通常コメントがブラウザ画面の左側に次々と表示され、スクロールしていくことを確認します。
5. Geminiの回答が生成された際に、右側にこれまで通りアニメーション付きで吹き出しが表示されることを確認します。
