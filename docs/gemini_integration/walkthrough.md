# Gemini連携機能 開発ウォークスルー

## 実施した内容

### 1. 設定情報の拡張 (`src/settings.rs`)
- Gemini APIとの連携に必要なフィールドを `Settings` 構造体に追加しました。
  - `gemini_api_key` (APIキー、文字列)
  - `gemini_enabled` (連携の有効化、真偽値)
  - `gemini_interval_secs` (自動要約・問い合わせインターバル秒数、デフォルト30秒)
  - `gemini_system_prompt` (AIへのシステム指示プロンプト、デフォルト値あり)
- アプリ初期化時の安全のため、`Default` トレイトを手動で再実装し、Gemini固有のデフォルト値を割り当てました。

### 2. Gemini API 連携モジュールの新規作成 (`src/gemini.rs`)
- Google AI Studioの `gemini-1.5-flash:generateContent` エンドポイントを非同期で呼び出す `query_gemini` 関数を実装しました。
- 複数コメント（結合テキスト）とシステムプロンプトを適切に JSON ペイロードに整形して送信し、返答されたマークダウン/テキストをパースして抽出する処理を構築しました。
- APIキーが空の場合のバリデーションや、API呼び出しエラー発生時のエラーハンドリングを整備しました。
- 空のAPIキーでの動作を確認するための単体テストを追加しました。

### 3. コメント制御ロジックの統合とポーリング (`src/app.rs`)
- 前回送信時以降に取得された新規コメントの差分を検知するロジック（`last_gemini_sent_comment_id` による判定）を実装しました。
- `app.rs` の `update` メソッド内でインターバル時間と新規コメントの有無を監視し、条件を満たした際に非同期でGeminiへリクエストを投げるトリガー処理を組み込みました。
- 問い合わせ中に連続して送信されることを防ぐフラグ制御（`is_waiting_gemini`）を実装しました。
- コメントが無い期間は送信をスキップし、その後コメントが到着したフレームで即座に動作する状態遷移を構築しました。
- Geminiからの回答が得られた際、それを専用の回答履歴構造体 `GeminiResponse` に蓄積し、同時に CeVIO AI 読み上げキューに送信して読み上げを発生させるロジックを統合しました。

### 4. UIの追加と微調整
- **左パネル（設定）**:
  - 「Gemini 連携」のチェックボックスを追加。
  - パスワード伏せ字（`•`）表示のAPIキー入力欄を追加。
  - 10秒〜300秒のインターバル時間設定スライダーを追加。
  - AIの口調やキャラクター、文字数上限などを制御できるシステムプロンプトの複数行入力欄を追加。
- **右パネル（Geminiの回答）**:
  - Geminiが有効（`gemini_enabled == true`）の場合のみ出現する、右側分割パネル `SidePanel::right("gemini_panel")` を追加。
  - タイムスタンプと「要約元のコメントプレビュー」、およびAIが生成した回答をチャット風の吹き出し枠（Group）で時系列表示する画面を実装。
  - 新しい回答が到着した際に、自動で末尾にスクロールする `gemini_scroll_to_bottom` 処理を追加しました。

### 5. 接続時の過去コメント読み込み防止とモデル名変更機能（追加対策）
- **過去コメントの読み込み防止 (`src/youtube.rs`)**:
  - 配信接続時に、すでに投稿されていた直近の既存コメントが大量に読み上げられるのを防ぐため、ポーリングループの初回取得コメントは `seen_ids` への登録のみに留め、UIスレッドへの送信をスキップするようにしました。これにより、接続後に新しく投稿されたコメントのみが正確に読み上げられます。
- **モデル名変更の設定項目化 (`src/settings.rs`, `src/gemini.rs`, `src/app.rs`)**:
  - Gemini APIの `v1beta` エンドポイントにおいて `gemini-1.5-flash` が廃止（404 Not Found）になったトラブルに対応し、デフォルトモデルを `gemini-2.5-flash` に更新しました。
  - 左側設定パネルにモデル選択用のドロップダウン（ComboBox）を追加し、主要なFlash/Flash-Liteモデルを簡単に選択できるようにしました（「手動入力」オプションを選択することで、任意のカスタムモデル名を入力することも可能です）。
  - リリース済みのFlash系主要モデル（`gemini-2.5-flash`、`gemini-2.5-flash-lite`、`gemini-2.0-flash` ,`gemini-1.5-flash`、`gemini-1.5-flash-8b`）を選択肢に登録し、存在しない架空のモデル名（`gemini-3.5-flash`）は選択肢から除外しました。
- **配信タイトルをプロンプトに統合 (`src/app.rs`)**:
  - 配信の内容（テーマやゲームなど）をGeminiが推論しやすくするため、接続時に取得した「YouTube配信のタイトル」をプロンプトの先頭に自動で付与して送信するようにしました。これにより、現在の配信トピックに沿った、より的確で盛り上がる回答が生成されます。
- **429 Too Many Requests 対策（リトライ処理の導入） (`src/gemini.rs`)**:
  - APIの利用制限に達した際、一時的なクォータ制限（RPM制限など）によるエラーを自動回避するため、指数バックオフ付き（3秒から開始し失敗毎に倍加）のリトライ処理を最大3回行うロジックを実装しました。
- **送信済みコメントの可視化（案1 ＆ 案3のハイブリッド導入） (`src/app.rs`)**:
  - 送信済みコメントと新着コメントを直感的に識別できるよう、以前のアイコン（案2）を廃止し、以下のデザインを統合しました。
  - **送信済みコメントのグレーアウト (案1)**: 送信済みコメントは、タイムスタンプ、ユーザー名、メッセージ本文、メモアイコンの一連の要素を一律で暗いトーン（グレー）にトーンダウンさせ、未送信コメント（明るい色）と明確に区別します。これにより、時刻の開始位置もきれいに揃った状態を維持できます。
  - **境界線の描画 (案1)**: 送信済みコメントと未送信コメントの間に **`─── 🤖 ここまでGemini送信済み ───`** の区切り線（薄い紫色の水平ライン）を動的に挟み込みます。
  - **未送信件数のリアルタイム表示 (案3)**: コメントログのヘッダーにある件数表示部分を拡張し、現在の総コメント数に加えて `(未送信: ○件)` の数値を常時表示させます。
