## [2026-07-09] GitHub Actions による自動リリースワークフローの導入

### 変更内容
- **.github/workflows/release.yml**: リリースタグ (`v*`) のプッシュをトリガーに、Windows環境でアプリをリリースビルドし、生成物を zip 圧縮して GitHub Release に自動デプロイする CI/CD ワークフローを新規構築。
- **Plan.md**: 自動リリース導入計画を追記。

### 変更理由
- バージョンタグをプッシュするだけで、実行ファイル（バイナリ）が自動ビルドされ、GitHub リリースページにアセットとしてアップロードされるようにし、手動リリースの手間を削減してミスを防ぐため。

### 影響範囲
- 今後 `v0.1.1` や `v0.2.0` といったタグをプッシュすると、GitHub Actions のランナー上でビルドが自動開始され、自動的にリリースノートが作成されてアセット（zip）が添付されます。

## [2026-07-09] ログ出力先の設定機能およびMarkdown形式（.md）への変更

### 変更内容
- **src/settings.rs**: `Settings` 構造体にログ保存先パスを保持する `log_dir` フィールドを追加。
- **src/app.rs (ログ処理部)**:
  - ログ保存先を動的に解決する `get_log_dir` 関数を追加。
  - `save_logs` 関数を修正し、ログ拡張子を `.md` に、形式をMarkdown（見出し・リスト）に変更。
  - `load_log_file` 関数を修正し、行のトリムとマークダウン強調表示の除去を追加することで、従来の `.txt` 形式と新しい `.md` 形式の両方を自動でパースして再生できるよう互換性を確保。
- **src/app.rs (UI部)**:
  - 「その他」設定タブに「📂 ログ保存先フォルダ」セクションを追加。現在の設定フォルダのパス表示と、フォルダ選択を行うための「変更」ボタン、および初期状態に戻す「初期化」ボタンを実装。

### 変更理由
- ログの保存先を任意のフォルダに変更したいというユーザーの要望、および保存したログをマークダウン形式で管理・閲覧したいという要望に対応するため。

### 影響範囲
- ログファイルが `.md` として保存されるようになり、設定UIから保存先を任意に変更できます。過去の `.txt` ログも引き続き読み込み可能です。
